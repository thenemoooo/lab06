# Лаборатор 6: Удамшил ба Бүрдмэл класс даалгаврыг амжилттай хийж гүйцэтгэв.

Даалгавар 3: Харьцуулалт ба Дүгнэлт

1. SortedIntList-ийн хэрэгжилтийн нарийн зүйлс дээр аль нь илүү хамааралтай вэ: composition (delegation) эсвэл inheritance?
Inheritance (Удамшил) нь илүү хамааралтай. Учир нь InheritanceSortedIntList нь SortedIntList-ийн шууд дэд класс бөгөөд эцэг классын дотоод логик, аргуудын хэрэгжүүлэлтээс шууд хамаардаг. Харин Composition нь зөвхөн интерфэйсээр дамжуулан харьцдаг тул холболт (coupling) багатай байдаг.
2. Хэрэв SortedIntList-ийн add аргыг ихээхэн өөрчилж эсвэл үйлдэл нь өөрчлөгдөх бол аль хэрэгжилт илүү их алдаа гаргах магадлалтай вэ?
Хариулт: Inheritance (Удамшил) илүү их алдаа гаргах магадлалтай. Жишээ нь: Хэрэв эцэг класс (SortedIntList) нь өөрийн addAll функц дотроо add функцийг дууддаг болж өөрчлөгдвөл, удамшсан классын тоолуур (totalAdded) хоёр дахин нэмэгдэх (double counting) аюултай. Composition-д ийм асуудал үүсэхгүй.
3. Inheritance ашиглахад үүссэн асуудлыг composition ашиглан ямар асуудлыг шийдвэрлэж чадна вэ?
Composition нь "Fragile Base Class" буюу эцэг классын өөрчлөлт дэд классыг эвдэх асуудлыг шийддэг. Энэ нь эцэг классын дотоод хэрэгжүүлэлтийн нарийн ширийнийг мэдэх шаардлагагүйгээр, зөвхөн интерфэйсээр дамжуулан үйлдлийг төлөөлүүлэн гүйцэтгэдэг (Delegation) тул илүү аюулгүй, уян хатан байдаг.
4. Өгөгдсөн хэрэгжилтийн үндсэн дээр, аль үед inheritance-г, аль үед composition-г ашиглах нь илүү зохистой вэ?
Хариулт: * Inheritance: Хэрэв хоёр классын хооронд жинхэнэ "is-a" (мөн мөн) харьцаа байгаа бөгөөд эцэг класс нь тогтвортой, өөрчлөгдөхгүй үед ашиглахад тохиромжтой.
Composition: Хэрэв "has-a" (агуулдаг) харьцаатай эсвэл системийн хэсгүүдийн хоорондын хамаарлыг багасгаж, уян хатан байдлыг хангах шаардлагатай бол ашиглах нь илүү зохистой.
